# Internal Documents

`internal/` 目录只承载项目组内部资料，不作为对外公开文档入口。

## 目录边界

- `docs/`：严格用于对外公开文档、公开使用说明、公开架构与公开维护说明
- `internal/`：用于项目组内部治理、Agent 角色设计、内部流程、候选方案、项目管理资料与未准备公开的工作台材料

## 当前分层约定

- `internal/agent-team/`：与项目组内 Agent 角色、治理方式、维护节奏、协作边界有关的内部文档
- 后续如有需要，可继续增加更细的内部子目录，但应保持职责清晰，不与 `docs/` 混放

## 规则

1. 任何面向项目用户、贡献者或外部维护者必须阅读的内容，优先放在 `docs/`
2. 与 `guardian-chief`、内部治理、项目工作台、Agent 协作方式有关的文档，放在 `internal/`
3. 若某份内部文档未来要公开，应重新整理并迁移到 `docs/`，而不是直接把内部材料暴露出去
