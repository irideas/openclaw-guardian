# Discussions 推荐帖头模板

本文档提供 `openclaw-guardian` 当前四个核心 Discussions 的推荐帖头模板，目标是降低记录成本，并保持结构一致。

使用原则：

- 模板是推荐结构，不要求每次逐项填满
- 应优先保留“判断 + 依据 + 下一步”三段
- 默认使用简体中文
- 如果一条记录已经超出当前 Discussion 的边界，应改投到更合适的载体

## 1. guardian-chief 工作台

适用场景：

- 记录一轮维护动作
- 同步工作方式变化
- 说明某次治理收口

推荐模板：

```md
## 维护记录：<本轮动作标题>

### 本轮判断
- <一句话说明这轮动作的主要判断>

### 已完成
1. <动作 1>
2. <动作 2>
3. <动作 3>

### 原因 / 背景
- <为什么现在做这件事>
- <它解决了什么边界或协作问题>

### 当前结果
- <当前已经明确的收口结果>

### 下一步
- <下一步建议 1>
- <下一步建议 2>
```

## 2. 候选问题管道 / candidate pipeline

适用场景：

- 归并一条新问题线索
- 更新某个候选现象的判断
- 记录继续观察或放弃理由

推荐模板：

```md
## 候选问题：<候选标题>

### 当前判断
- <继续观察 / 形成候选 / 放弃 / 等待上游>

### 现象摘要
- <用户可见现象>
- <触发命令或触发场景>
- <已知影响范围>

### 来源
- <聊天 / 日志 / 社区 / 上游 issue / 本机复现>

### 已知事实
- <事实 1>
- <事实 2>
- <事实 3>

### 未确认点
- <仍需确认的问题>

### 与既有项关系
- <是否与已有候选或 issue 重合>

### 建议下一步
- <继续收证>
- <准备 issue 草稿>
- <等待更多样本>
```

## 3. Owner 决策记录 / owner decisions

适用场景：

- 提交需要 Owner 拍板的事项
- 留痕最终决策结果

推荐模板：

```md
## 决策项：<决策标题>

### 需要决策的问题
- <这次到底要拍板什么>

### 背景
- <问题背景>
- <为什么现在需要决策>

### 备选项
1. <方案 A>
2. <方案 B>
3. <方案 C>

### 当前建议
- <agent 当前更倾向的建议>
- <建议原因>

### 风险与代价
- <风险 1>
- <风险 2>

### Owner 决策
- <待补充 / 已决定：...>

### 后续动作
- <决策后要执行的动作>
```

## 4. 每周维护回顾 / weekly maintenance review

适用场景：

- 周节奏复盘
- 汇总线索、动作、进展与下周重点

推荐模板：

```md
## 周维护回顾：<日期范围>

### 本周判断
- <一句话总结本周最重要判断>

### 本周新增线索
- <线索 1>
- <线索 2>

### 候选问题进展
- <候选项 A 的进展>
- <候选项 B 的进展>

### 上游与社区变化
- <相关上游 issue / 社区案例>

### 本周已完成维护动作
- <文档 / 测试 / 治理 / 工作台调整>

### 需要 Owner 关注
- <需要确认的点>

### 下周建议
- <建议 1>
- <建议 2>
```

## 5. 最小写法

如果当下不适合写完整模板，至少保留这三段：

```md
### 判断
- <当前判断>

### 依据
- <关键事实>

### 下一步
- <建议动作>
```

这是最低限度的结构，优先保证可读、可追踪、可继续推进。
